# バディファイト試作アプリ 説明書

## 1. 起動方法

このアプリはカードデータをJSONから読み込むため、ローカルサーバー経由で開きます。

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\server.ps1 -Port 4173
```

起動後、ブラウザで以下を開きます。

- 対戦画面: `http://127.0.0.1:4173/index.html`
- デッキ構築画面: `http://127.0.0.1:4173/builder.html`

ネット対戦を使う場合は、通常の `server.ps1` ではなく以下を起動します。

```powershell
node .\netplay-server.js --port 4173
```

起動後、ブラウザで `http://127.0.0.1:4173/netplay.html` を開きます。LAN内で別PCから参加する場合は `--host 0.0.0.0` を付けて起動し、ホストPCのIPアドレスにアクセスします。

同一ネットワーク外の相手と対戦する場合は、Node.js対応の公開サーバーに配置して `npm start` で起動します。公開URLの `/netplay.html` を開けば、部屋番号または参加URLで入れます。公開手順は `PUBLIC_DEPLOY.md` を参照してください。

## 2. 対戦画面

画面上部の `1P` と `2P` で使用デッキを選び、`新規` を押すと新しいファイトを開始します。

基本操作はフェイズ順に行います。

1. ドロー
2. チャージ&ドロー、またはメインへ
3. コール、装備、魔法使用、起動能力使用
4. アタックへ
5. 攻撃、連携攻撃、対抗、攻撃解決
6. ファイナルへ
7. 必殺技、攻撃、ターン終了

カードにカーソルを合わせると詳細ポップアップが表示されます。

## 3. ネット対戦画面

`netplay.html` は2つのブラウザから同じ部屋に入って対戦します。

1. ホスト側が `部屋作成` を押します。
2. 表示された部屋番号、または `URLコピー` でコピーしたURLを相手に渡します。
3. 参加側は部屋番号を入力して `参加` を押します。
4. 1Pは1Pデッキ、2Pは2Pデッキだけを変更できます。
5. 1P側が `新規` を押すとファイトが同期開始されます。

ネット対戦では、手札欄には自分の席の手札だけが表示されます。相手の手札は枚数と裏面だけが見えます。

## 4. デッキ構築画面

`builder.html` では、収録済みカードからデッキを作れます。

- `デッキ名`: 保存・エクスポート時の名前
- `フラッグ`: 使用ワールド
- `バディ`: ファイト中にバディコールできるカード
- `カード検索`: カード名、カード番号、属性、製品などで検索
- `追加`: メインデッキに1枚追加
- `減らす`: メインデッキから1枚減らす
- `フラッグ`: 選択カードをフラッグに設定
- `バディ`: 選択カードをバディに設定
- `整列`: 採用カード一覧を種類とカード番号順に並べ替え

## 5. デッキ構築ルール

ビルダーは以下を検証します。

- フラッグを1枚選ぶ。
- バディを1枚選ぶ。
- バディとフラッグはメインデッキ枚数に含めない。
- メインデッキは50枚以上。
- 同じカード名は4枚まで。
- フラッグ条件外のワールド/属性/種類のカードはデッキに入れられるが、ファイト中は使えないため警告する。

フラッグは共通定義から選びます。同じ名前のフラッグが製品内に収録されていても、アプリでは重複登録せず共通フラッグとして扱います。

警告はデッキ構築の注意表示です。対戦時は、選択中のフラッグで使えないカードの使用操作を止めます。

## 6. 保存

`保存` を押すと、作成中のデッキがブラウザ内に保存されます。

保存したデッキは以下で使えます。

- デッキ構築画面の `保存済みデッキ` から読込
- 対戦画面の `1P` / `2P` デッキ選択

保存先はブラウザのローカルストレージです。別ブラウザや別PCには自動では移りません。

## 7. エクスポート

`エクスポート` を押すと、デッキJSONがテキスト欄に出力されます。

可能な環境ではクリップボードにもコピーします。コピーできない環境では、テキスト欄が選択状態になるので、手動でコピーしてください。

ファイルとして保存したい場合は `JSON保存` を押します。

## 8. インポート

デッキJSONをテキスト欄に貼り付けて `インポート` を押すと、そのデッキを読み込みます。

読み込んだだけでは保存済みデッキには入りません。対戦画面でも使いたい場合は、読み込み後に `保存` を押してください。

## 9. 対戦診断ログ

対戦画面の `ログ保存` を押すと、対戦中の操作・選択・解決結果をJSONで保存します。

このログには、手札、デッキ順、選択肢、選んだカード、未実装ログ、最終盤面などのデバッグ用情報が含まれます。問題の起きた直後に保存すると、未実装カードや想定外の処理を追いやすくなります。

## 10. 配布用フォルダ

外部に渡す場合は `buddyfight-app-export` フォルダを使います。

このフォルダには、起動に必要なファイルだけが入っています。

- `index.html`
- `netplay.html`
- `builder.html`
- `styles.css`
- `builder.css`
- `src/`（対戦エンジン本体。`01-foundation.js`〜`21-bootstrap.js` の21モジュール一式。フォルダごと同梱する）
- `builder.js`
- `netplay-server.js`
- `package.json`
- `PUBLIC_DEPLOY.md`
- `server.ps1`
- `README.md`
- `MANUAL.md`
- `data/`

Edgeプロファイル、スクリーンショット、PDF変換画像などは不要です。

## 11. 注意

このアプリはローカル対戦用の試作です。カード効果や裁定は順次実装中のため、未対応の効果や簡略化された処理が残る場合があります。

ネット対戦は初期実装です。UI上は相手の手札を表示しませんが、現段階では盤面同期を優先しており、競技用の完全な不正耐性を持つ権威サーバー方式ではありません。
