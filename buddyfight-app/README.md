# バディファイト試作アプリ

「フューチャーカード バディファイト」をベースにした、ローカル対戦 / ネット対戦用のWebプロトタイプです。カード画像は使っていません。

ルールは2018年6月以前、つまり『フューチャーカード 神バディファイト』開始前のものを基準にしています。神バディファイト以降の追加要素や裁定変更は採用しません。

> このREADMEが唯一の説明書です（旧 MANUAL.md / はじめにお読みください.txt は本ファイルに統合しました）。

---

## かんたん起動（Windows・ダブルクリック）

- **`起動.bat`** … ローカル対戦・デッキ構築（PowerShell静的サーバ、Node.js不要）。黒いウィンドウが開き、ブラウザで `index.html` が開きます。終了は黒いウィンドウを閉じます。
- **`権威起動.bat`** … オンライン対戦（権威サーバ版・手札秘匿/観戦／**Node.js が必要**）。ブラウザで `play.html` が開きます。

注意:
- カードデータをサーバー経由で読み込むため、**HTMLファイルを直接ダブルクリックしても動きません**。必ず bat（またはサーバー）経由で開いてください。
- 「Windows によって PC が保護されました」と出たら「詳細情報」→「実行」。
- 画面右上の ☀/🌙 ボタンでライト/ダークの配色を切り替えられます（次回も保持）。

## 起動（コマンドで起動する場合）

カードデータをJSONから読み込むため、ローカルサーバー経由で開きます。

### ローカル対戦 / デッキ構築

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\server.ps1 -Port 4173
```

- 対戦画面: `http://127.0.0.1:4173/index.html`
- デッキ構築: `http://127.0.0.1:4173/builder.html`

### ネット対戦（権威サーバ版・手札秘匿/観戦）

サーバー側でゲームエンジンを実行し、各プレイヤーには自分視点（自分の手札・山札だけが見える）を配信する権威サーバ方式です。**相手のブラウザへ手札・山札が渡りません**。観戦者は両者の手札も見られます。

Windows なら `権威起動.bat` をダブルクリック（Node.js が必要）。または:

```powershell
npm run start:auth:local
```

`http://127.0.0.1:4174/play.html` を開きます（`権威起動.bat` は自動でブラウザを開きます。ポートが予約済み/使用中なら自動で別ポートに切り替わるので、実URLはサーバー窓の表示を参照）。「部屋作成」で部屋番号を作り、相手は同じ画面で番号を入れて「参加」。先手 / 後手 / 観戦は参加後に切り替えられます（参加順に縛られません）。両席が埋まったら「対戦開始」。LAN公開は `npm run start:auth`（`--host 0.0.0.0`）。

対応操作は **ドロー / フェイズ進行 / ターン終了 / チャージ / コール / 使用 / 攻撃 / 防御（対抗）/ 解決（＝対抗で何も使わずパス）**。加えて **選択ダイアログ・任意能力ゲート・じゃんけん・ソウルガード等の割り込みは prompt 往復で対応**します。**自作デッキの持ち込み**（builder と共有の `localStorage` から選択して送信）と、**対抗ウィンドウでの「打てる手」表示**にも対応。設計は `docs/オンライン対戦権威サーバ設計_2026-06-25.md` を参照。

### インターネット公開（常時起動ホストへデプロイ）

常時起動できる Node.js ホストにこのフォルダを置けば、ネット越しの相手と部屋番号で対戦できます。**依存パッケージはゼロ（バニラ）**なので `npm install` 不要、必要なのは **Node.js 18 以上**だけ。手順の詳細・プロキシ設定・永続化は `PUBLIC_DEPLOY.md` を参照。

- 起動コマンド: `npm run start:auth`（＝ `node server/authoritative-server.js --host 0.0.0.0`。`start:public` も同じ）
- ヘルスチェック: `/healthz`（`{ok:true,rooms:N}` を返す）
- ポート: 環境変数 `PORT` を自動採用（無ければ 4174）。`HOST` も env 対応
- 公開後に開くURL: `https://<公開ドメイン>/play.html`

同梱の設定ファイルでそのままデプロイできます:

- **Fly.io**: `fly launch`（同梱 `fly.toml` を自動検出）→ `fly deploy`
- **Railway / Render / Heroku 系**: 同梱 `Procfile`（`web: node server/authoritative-server.js --host 0.0.0.0`）。`PORT` は自動注入
- **Docker / VPS**: 同梱 `Dockerfile`（`node:20-slim` ＋ `/healthz` HEALTHCHECK）。pm2 例 → `PORT=8080 AUTH_DATA_DIR=/var/lib/buddyfight-auth pm2 start "npm run start:auth"`

公開時の注意:

- リバースプロキシ／CDN では **SSE のため response buffering を無効化**すること（Nginx: `proxy_buffering off;`。サーバは `X-Accel-Buffering: no` を送出済だがプロキシ側でも要設定）。
- 再起動を越えて進行中の局面を復元するなら**永続ディスクをマウント**し `AUTH_DATA_DIR` をそのパスへ。トークンを含むので **web root の外**に置く（サーバ側にも静的配信 deny の二重防御あり）。
- 再接続: リロード／瞬断しても `localStorage` の席トークンで自動復帰（手動操作不要）。相手切断中は「待機中」を表示。

> 実ホストでの動作確認は各自のデプロイ環境で行ってください（headless スモーク／e2e はローカルで緑。本番起動はユーザー側）。

## 画面の使い方

### 対戦画面（index.html）

上部の `1P` / `2P` で使用デッキを選び、`新規` で新しいファイトを開始します。基本はフェイズ順:

1. ドロー
2. チャージ&ドロー、またはメインへ
3. コール、装備、魔法使用、起動能力使用
4. アタックへ
5. 攻撃、連携攻撃、対抗、攻撃解決
6. ファイナルへ
7. 必殺技、攻撃、ターン終了

カードにカーソルを合わせると詳細ポップアップが出ます（スマホはタップでカードシート）。

### ネット対戦（play.html・権威サーバ版）

`権威起動.bat`（または `npm run start:auth:local`）で起動します。既定ポートは 4174 で、予約済み/使用中なら自動で別ポートに切り替わります（実URLはサーバー窓に表示。`権威起動.bat` はそのポートで自動的にブラウザを開きます）。部屋作成→相手が番号で参加→役割（先手/後手/観戦）を選んで「対戦開始」。手札・場のカードをクリックすると操作メニュー、攻撃は対象（相手のカード/本体）をクリックして確定します。

### デッキ構築（builder.html）

収録済みカードからデッキを作れます。

- `デッキ名`: 保存・エクスポート時の名前
- `フラッグ`: 使用ワールド
- `バディ`: ファイト中にバディコールできるカード
- `カード検索`: カード名 / 番号 / 属性 / 製品などで検索
- `追加` / `減らす`: メインデッキの増減
- `フラッグ` / `バディ`: 選択カードをフラッグ/バディに設定
- `整列`: 採用カードを種類・カード番号順に並べ替え

#### デッキ構築ルール（ビルダーの検証）

- フラッグを1枚、バディを1枚選ぶ（どちらもメインデッキ枚数に含めない）。
- メインデッキは50枚以上。同じカード名は4枚まで。
- フラッグ条件外（ワールド/属性/種類）のカードも入れられるが、ファイト中は使えないため警告。
- フラッグは共通定義から選ぶ（同名フラッグが製品内にあっても重複登録せず共通フラッグ扱い）。
- 対戦時は、選択中のフラッグで使えないカードの使用操作を止めます。

#### 保存 / エクスポート / インポート

- `保存`: 作成中デッキをブラウザ内（ローカルストレージ）に保存。デッキ構築の `保存済みデッキ` 読込、対戦画面の `1P`/`2P` 選択で使えます。別ブラウザ・別PCには自動では移りません。
- `エクスポート`: デッキJSONをテキスト欄へ出力（可能ならクリップボードにもコピー）。`JSON保存` でファイル保存。
- `インポート`: デッキJSONを貼り付けて読込。対戦でも使うなら読込後に `保存` を押す。

### 対戦診断ログ

対戦画面の `ログ保存` で、操作・選択・解決結果をJSONで保存します。手札・デッキ順・選択肢・選んだカード・未実装ログ・最終盤面などのデバッグ情報を含みます。問題発生直後に保存すると原因を追いやすくなります。

## 入っている機能

- 画面文言の日本語化 / 2018年6月以前ルールへの固定 / 詳細ルール ver.2.05 のフェイズ進行
- 公式スタートデッキ・トライアルデッキの製品別データ / デッキ構築ツール
- 権威サーバ版オンライン対戦（部屋番号方式・手札/山札の秘匿・観戦・役割変更）
- 操作・選択・解決結果を含む対戦診断ログのJSON保存
- カード検索、デッキ検証、インポート / エクスポート、作成デッキのブラウザ保存と対戦画面での選択
- 共通フラッグ定義とフラッグ別の使用可能カード制限
- デッキ / 手札 / ゲージ / ドロップ / ライフ、レフト / センター / ライト / アイテムの場
- ドロー、チャージ&ドロー、コール、バディコール、装備、魔法、必殺技、攻撃、攻撃解決、ターン終了
- 攻撃中の攻撃側 / 防御側手札切替、先攻1ターン目のドローなし・攻撃1回まで、【対抗】を各ファイター1回まで
- メイン → アタック → ファイナル → ターン終了、ファイナル中の攻撃・必殺技、連携攻撃（合計攻撃力/打撃力で解決）
- サイズ合計3制限、同エリア再コール時のドロップ処理、デッキ0枚敗北、センター中の本体攻撃不可
- カード詳細ポップアップ / カードシート、対象を選ぶ効果のプレイヤー選択、攻撃無効化時の連携全終了
- カードデータとデッキレシピを製品別JSONに分離、複雑なコスト/効果のJSONルール定義
- 逆天、着任、ソウルガードを追加カード側から使えるエンジン土台
- スマホ対応（レスポンシブ・タッチ操作・ライト/ダーク2モード）

## 配布用フォルダ（同梱ファイル）

外部に渡すときは `buddyfight-app-export` フォルダ（起動に必要なファイルだけ）を使います。

- `index.html`（ローカル対戦）/ `play.html`（権威サーバ版ネット対戦）/ `builder.html`（`netplay.html` は旧・中継版で現在は使用しません）
- `styles.css` / `builder.css`
- `src/`（対戦エンジン本体。`01-foundation.js`〜`21-bootstrap.js` の21モジュール。**フォルダごと**同梱）
- `builder.js` / `play.js`（権威用シンクライアント）
- `server/`（権威サーバ。`authoritative-server.js` ＋ `engine-host.js`。**フォルダごと**同梱）
- `netplay-server.js` / `server.ps1` / `package.json`
- `起動.bat`（ローカル対戦・PowerShell・Node不要）/ `権威起動.bat`（権威版ネット対戦・Node必須）
- `README.md`（本ファイル）/ `PUBLIC_DEPLOY.md`
- `data/`（カード・デッキ・フラッグのJSON）

`tests/` `docs/` `e2e/` `*.smoke.js` などの開発物、Edgeプロファイル、スクリーンショット、PDFは不要です。

## データについて

カード名、枚数、数値、カード種別はブシロード公式の日本語カードリストを参考にしています。公式カードテキストとフレーバーテキストの全文はアプリ内に転載せず、効果はアプリ内で処理している内容に合わせたテキストとして表示します。

カード追加では、基本情報は `data/cards/<product-id>.json`、構築済み/テストデッキは `data/decks/<product-id>.json` に追加し、`data/cardsets.json` / `data/decksets.json` に登録します。フラッグは `data/flags.json` の共通定義で管理し、製品別カードJSON内のフラッグは読み込み時に除外します。よくある処理は `cost` / `conditions` / `target` / `effects` のJSON命令で表現し、個別処理が必要なカードだけ `handler` に逃がす設計です。

ルール確認では、ユーザー提供の `buddyfight_rule_ver205.pdf` と、2018年6月2日適用のルール変更が明記された公式Q&Aを参照し、その変更前の挙動に合わせています。

確認元：

- https://fc-buddyfight.com/products/bfsd01
- https://fc-buddyfight.com/cardlist/
- https://fc-buddyfight.com/wp/wp-content/uploads/buddyfight_rule_ver204.pdf
- https://fc-buddyfight.com/rules/question/search/?page=313
